Main Structure

This theme has a responsive layout with 12 columns. You can customize the content, sidebar, navbar to be as width as you want. The general template structure is the same throughout the template. Here it is:

<section id="header" class="container">
<header id="logo" class="grid3 col">
<hgroup>
<h1><a href="index.html"><img src="img/logo.png" class="scale" width="180" height="67" alt="CleanGold"/></a></h1>
</hgroup>
</header>
<nav id="menu" class="grid9 col clearfix">
...
</nav>
<div class="sep sep-small"></div>
<span id="switch">Menu <strong>&#8801;</strong></span>
</section>
<section id="content" class="container">
...
<a href="#top" id="top-link"><span></span></a>
</section>
<footer id="footer">
...
</footer>

You can change the logo image by placing an image file into the img directory and setting it in the <h1> element.

The template contain <head> information that you would probably want to edit: title, description (SEO relevant), keywords (SEO relevant), favicon, apple touch icon and Facebook share image:

<title>Title goes here</title>
<meta name="description" content="Description entered for SEO" />
<meta name="keywords" content="list of keywords" />
<link rel="shortcut icon" href="favicon.png" />
<link rel="apple-touch-icon" href="img/touch-icon.png" />
<link rel="image_src" href="img/touch-icon.png" />

If you would like to edit the color, font, or style of any elements in one of these columns, you would do the following:

a, a:visited { color: #333; }
a:hover, a:focus { color: #000; }

If you find that your new style is not overriding, it is most likely because of a specificity problem. Scroll down in your CSS file and make sure that there isn't a similar style that has more weight.

.container a { color: #555; }
.container a:hover, .container a:focus { color: #000;  }

So, to ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours.

 

Grid Layout

The 12-column grid is divided into portions that are fixed wide. Each column has margin on the left and right, which create some gutters between columns. The columns scale down well on mobile and can be used to create a layout with ease. There is a specific set of class based rules you need to follow. To get the idea please read a few information from this article: http://sonspring.com/journal/960-grid-system

The container element is the top level DOM element that needs to be present. All the main section of the site already have this class, ex:

<section id="header" class="container">

After this there are column sections identified by the col class. Each column has a specific gridX class to specify the dimensions. For example, to build a content area 2/3 width and a sidebar 1/3 layout you'll have to follow this markup:

 

<div class="grid9 col">
...
</div>
<div class="grid3 col">
...
</div>
<div class="clear"></div>

A row must be 12 columns width, meaning if you add all the column sizes it must be equal to 12 (in our example 9+3=12). The clear element is used for floating fixes. Use it when neccesary: http://css-tricks.com/containers-dont-clear-floats/

Please see the grids.html file as a starting point for your grid layout.

CSS Styling

I'm using only one highly optimized CSS file that uses mobile first approach. That means all the style is applied for mobile devices first and if the screen is bigger the template has more rich, bandwidth consuming features. The stylesheet contains a general reset CSS file so all browsers behave alsmost the same way. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden at any time.

The second file contains all of the specific stylings for the page. The file is separated into sections using:

/* Section */
some code
/* Footer */
some codeetc, etc.

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited. Almost every widget in the template can be customized easily this way. Make sure you check out the HTML markup first and then modify its style in the CSS file.

JavaScript

This theme imports three mandatory Javascript files.

  1. jQuery
  2. Minified plugins
  3. The sites initialization methods
  1. jQuery is a Javascript library that greatly reduces the amount of code that you must write.
  2. The template includes a rich set of plugins in order to enhace the user experience: window smartresize,  mousewheel support, easing effects, tabify, Quicksand, fancyBox 2, Infinite Scroll 2, FlexSlider, jquery.event.drag + jquery.event.drop, Roundabout, panorama360, Isotope.
  3. The site.js script is used to initialize all the plugins and add events to certain objects. Here is the place where you can modify all the widgets behaviour.

 

Here are the default setting of the plugins:
$('.flexslider.simple').flexslider({
   animation      : "slide",
   animationSpeed : 750,
   easing         : "easeInOutCirc",
   useCSS         : false,
   controlNav     : false,
   start          : function() {
      $('html').removeClass('loading');
   }
});

 

$('.roundabout ul').roundabout({
   responsive    : true,
   enableDrag    : true,
   easing        : "easeInOutCirc",
   dropEasing    : 'easeOutBounce',
   minScale      : 0.6,
   duration      : 400,
   dropDuration  : 500
});

 

$('.panorama-view').panorama360({
   sliding_direction   : 1,
   sliding_controls    : true,
   sliding_interval    : 20
});

 

$(".lightbox, .button-fullsize, .fullsize").fancybox({
   padding    : 0,
   loop       : true,
   fitToView  : true,
   mouseWheel : false,
   width      : '100%',
   height     : '90%',
   autoSize   : false,
   closeClick : false,
   overlay    : { showEarly  : true },
   helpers    : { media : {} }
});

 

$('.isotope').isotope({
   itemSelector    : '.col:visible',
   animationEngine : 'jquery',
   resizable       : true
});

Make sure you check out thre documentation of each plugin for its complete set of options.

Tips & Tricks

Here are a couple of neat tricks used in this theme and how to use them:

  1. set active menu item: add a current_page_item class to the active menu element (add it to the <li> element);
  2. create submenu: add a arrow class to the parent menu element and specify a <ul class="sub-menu"> list of elements to be used for the submenu (see markup for details);
  3. the title of each page is a <h1> marked with page-title class; you can also specify a description of the page in a <div class="subtitle"> container, but this is optional;
  4. add a tooltip to an element: add tooltip class to the element and specify a tip attribute: <a href="#" class="tooltip" tip="Tooltip example">link</a>
  5. set image to be responsive: just add a scale class to the image: <img src="thumb.jpg" class="scale" alt="" />
  6. make an image open in a lightbox: add a lightbox class to the wrapping link and everything should work out well: <a href="full-image.jpg" class="lightbox"><img src="thumb.jpg" alt="" /></a>
  7. create a thumbnail with link or fullsize icons: wrap your image in a div.thumb container and add extra information to it like this: 
    <div class="thumb">
    <a href="link.html" class="lightbox"><img src="thumb.jpg" class="scale" alt="" /></a>
    <div class="info pattern">
        <a href="link.html" class="button-link"></a>
    </div>
</div>
    Additional information details can be added, please see the index.html as a starting point example;
  8. add cool CSS3 animation on any element hover: there are plenty of animations to choose from: shake, flash, bounce, tada, swing, wobble, wiggle, pulse, flip. For example, if you want a submit buttons to shake when hovered specify the “animated” class and the animation type name: <input type="submit" class="animated shake" value="Go" />
  9. sticky footer: sometimes there are problems with pages that have little content, especially with the footer. It should stick to the bottom of the page. To add this functionality follow the example from the 404.html file; the markup with the “wrapper-frame”, “wrapper-row”, “wrapper-row wrapper-expand” classes. The original solution can be found here:http://pixelsvsbytes.com/blog/2011/09/sticky-css-footers-the-flexible-way/ (slightly modified);

  10. make videos and embeds responsive: wrap the video or embed in the following markup structure:

    <div class="video-container">
    <div class="video-wrapper">
        <iframe src="..."></iframe>
    </div>
</div>
  11. there are multiple options when it comes to buttons: small, normal, big , yellow, gray, black. It's easy to add a .button class to any element (anchors for example) ad set a color preference and size option. Here is a big gray button: <a href="#" class="button big gray">Big Gray</a>. Please see more details in the forms-buttons.html file;
  12. alerts are a good way to capture attention: <div class="alert info">Information box</div>. Please see the alerts-widgets.html file for different options;
  13. accordions: add a accordion class to any <ul> list and it will be transformed automatically into an accordion widget; additionally you can specify which accordion box is opened initally with an active class on each list's <li>;
  14. tab container: wrap the content in a tab-container element with a <ul class"tabs"> list followed by .tab_content elements; please see the alerts-widgets.html for the markup;
